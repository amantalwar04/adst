# Linux Application Debugging with System Debugger 1. Launch SDK. 2. Create a Linux application. 3. Select the application you want to debug. 4. Select **Run \> Debug Configurations.** 5. Click **Xilinx C/C++ application \(System Debugger\)** to create a new configuration. ![](../images/linux_app_debugging_2.png) 6. In the Debug Configuration window: 1. Click the **Target Setup** tab. 2. From the **Debug Type** drop-down list, select **Linux Application Debug**. ![](../images/linux_app_debugging_3.png) 3. Provide the Linux host name or IP address in the **Host Name** field. ![](../images/linux_app_debugging_4.png) 4. By default, tcf-agent runs on the 1534 port on the Linux. If you are running tcf-agent on a different port, update the **Port** field with the correct port number. 5. In the Application Tab, click **Browse** and select the project name. SDK automatically fills the information in the application. ![](../images/linux_app_debugging_5.png) 6. In the **Remote File Path** field, specify the path where you want to download the application in Linux. ![](../images/linux_app_debugging_6.png) 7. If your application is expecting some arguments, specify them in the **Arguments** tab. ![](../images/linux_app_debugging_7.png) 8. If your application is expecting to set some environment variables, specify them in the **Environments** tab. ![](../images/linux_app_debugging_8.png) 9. Click the **Debug** button. A separate console automatically opens for process standard I/O operations. ![](../images/linux_app_debugging_9.png) ![](../images/linux_app_debugging_10.png) 10.
