# connect Connect to hw\_server/TCF agent. ## Syntax connect \[options\] Allows users to connect to a server, list connections or switch between connections. ## Options |Option|Description| |------|-----------| |-host |Name/IP address of the host machine| |-port |TCP port number| |-url |URL description of hw\_server/TCF agent| |-list|List open connections| |-set |Set active connection| |-new|Create a new connection, even one exist to the same url| |-xvc-url |Open Xilinx Virtual Cable connection| ## Returns The return value depends on the options used. `-port, -host, -url, -new`: `` of the new connection or error if the connection fails `-list`: list of open channels or nothing when there are no open channels `-set`: nothing ## Example\(s\) ``` connect -host localhost -port 3121 ``` Connect to hw\_server/TCF agent on host localhost and port 3121. ``` connect -url tcp:localhost:3121 ``` Identical to previous example. **Parent topic:** [Target Connection Management](../../xsct/connections/reference_xsct_connections.html) **Related information** [disconnect](../../xsct/connections/reference_connections_disconnect.html) [targets](../../xsct/connections/reference_connections_targets.html) [gdbremote connect](../../xsct/connections/reference_connections_gdbremote_connect.html) [gdbremote disconnect](../../xsct/connections/reference_connections_gdbremote_disconnect.html)
