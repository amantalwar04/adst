# After installing SDK on Ubuntu machines, toolchain paths are not accessible from SDK ## Issue After installing the SDK on Ubuntu machines, tool chain paths are not be accessible. You may get the following error while building the applications: ``` arm-xilinx-eabi-gcc: No such file or directory ``` ## Solution Need to instal necessary packages to access toolchian paths from SDK. To install the required packages, use the following command. ``` sudo apt-get install lib32z1 lib32ncurses5 lib32bz2-1.0 lib32stdc++6 ``` See [**Answer Record 63561**](http://www.xilinx.com/support/answers/63561.html) for additional information. **Parent topic:** [Known Issues](../SDK_references/sdk_knownissues.html)
